# TheEnd 项目开发路线图

本项目将采用增量开发的方式，逐步实现 TheEnd 音游的各项功能。

## 第一阶段：项目初始化与基础框架搭建

1.  **技术选型与环境配置**：
    *   使用 Vue.js 作为前端框架。
    *   配置开发环境，集成 Vite 或 Webpack 作为构建工具。
    *   初始化 Git 仓库，并创建 `.gitignore` 文件以忽略 `node_modules` 等无关文件。
2.  **基础页面结构**：
    *   创建主页 (`/src/views/Home.vue`) 作为游戏入口。
    *   创建游戏页面 (`/src/views/Game.vue`) 用于承载游戏核心逻辑。
    *   创建歌曲选择页面 (`/src/views/SongSelect.vue`)。
    *   配置 Vue Router 实现页面间的路由跳转。
3.  **静态资源管理**：
    *   创建 `/src/assets` 文件夹用于存放图片、音频等静态资源。
    *   引入一个临时的背景图片和音轨用于测试。

## 第二阶段：核心游戏机制实现

1.  **游戏场景绘制**：
    *   使用 HTML5 Canvas 或 SVG 绘制游戏背景、判定线和下落的音符。
    *   实现音符的下落动画。
2.  **音符类型**：
    *   实现短键 (Tap) 的基本逻辑。
    *   实现长键 (Hold) 的逻辑。
    *   实现滑键 (Drag) 的逻辑。
3.  **判定与得分**：
    *   实现音符与判定线的碰撞检测。
    *   根据判定结果（Perfect, Good, Miss）给出反馈。
    *   实现简单的计分系统和连击 (Combo) 统计。
4.  **音频同步**：
    *   实现游戏音轨的播放与暂停。
    *   确保音符下落与音乐节奏同步。

## 第三阶段：视觉效果与UI/UX优化

1.  **视觉风格实现**：
    *   根据 `TheEnd.md` 的描述，实现以线条、方块和辉光为主的视觉风格。
    *   为音符的点击、消失等事件添加粒子效果。
    *   设计并实现动态的背景效果。
2.  **UI界面完善**：
    *   设计并实现主菜单界面。
    *   实现歌曲选择界面的详细布局，包括铺面信息的展示。
    *   完善游戏内UI，包括得分、Combo、进度条等。
3.  **用户体验优化**：
    *   优化操作手感，确保响应灵敏。
    *   添加必要的过渡动画，使界面切换更自然。

## 第四阶段：铺面编辑器（自制铺）

1.  **编辑器基础功能**：
    *   创建一个新的页面作为铺面编辑器。
    *   实现铺面信息的导入（音频、封面等）。
    *   在时间轴上可视化地放置和编辑不同类型的音符。
2.  **高级编辑功能**：
    *   实现判定线的移动、旋转等动画效果的编辑。
    *   提供预览功能，方便创作者随时查看铺面效果。
3.  **铺面文件的导入与导出**：
    *   设计铺面文件格式（如 JSON），用于存储铺面数据。
    *   实现铺面文件的保存和读取功能。

## 第五阶段：高级功能与发布准备

1.  **设置页面**：
    *   实现音量调节、下落速度、背景亮度等设置选项。
2.  **性能优化**：
    *   对游戏进行性能分析和优化，确保在不同设备上流畅运行。
3.  **打包与部署**：
    *   将项目打包为静态文件。
    *   部署到网页服务器或GitHub Pages。

## 第六阶段：安卓版本移植

1.  **技术选型**：
    *   研究并选择合适的跨平台打包工具，如 Capacitor 或 Cordova。
2.  **移动端适配**：
    *   调整UI布局以适应不同尺寸的手机屏幕。
    *   优化触摸操作的响应。
3.  **打包与测试**：
    *   将网页应用打包为安卓 APK。
    *   在真实的安卓设备上进行测试和调试。
